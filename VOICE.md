# VOICE.md — Omar Al-Bakri writing voice

Reference this before writing any user-facing copy: README intros, landing pages, blog/newsletter posts, LinkedIn drafts, product descriptions, docs prose. Not for code comments or commit messages.

## The core move

Everything in this voice runs on **reframe, not hype**. State the conventional read, then invert it in one clean sentence. That inversion — not an adjective — is the hook.

> "Stablecoins are sold as a payments upgrade. In reality, they are a bid for the banking system's cheapest funding."

> "Most salespeople use AI to write more emails. The good ones use it to write fewer."

If a piece doesn't have a reframe, it doesn't have an opening yet.

## Ration the antithesis device — this is the failure mode to watch for

The twin-sentence antithesis ("X isn't Y. It's Z.") is the sharpest tool in this voice, which is exactly why it's the one that gets overused. It should appear **once per piece** — once per newsletter essay, once per landing page, once per README intro. Not once per section.

Real example of the failure: a page draft once shipped with this as literally every section headline, in order —

- "AI products fail at the handoff between commercial need and technical execution. I remove the handoff."
- "AI products are easy to describe. Working systems leave evidence."
- "The tools are not the differentiator. The judgement behind them is."
- "The engineering is new. The financial-services context is not."
- "An AI strategy deck is not a delivery model. Someone still has to own the system."

Any one of these lands as conviction. Five in a row on one scroll reads as a template — the reader's eye catches the rhythm by headline three and stops reading the words. **If you're about to write a third "X is not Y" headline on the same page, stop and write something else instead**, even something plainer. A flat, literal headline next to a few sharp ones still reads as confident; five sharp ones in the same shape reads as a mail-merge.

**Alternative headline shapes to reach for once the antithesis has been used once on a page:**
- A plain factual statement, no contrast: "Selected work." "How I work." "What I can own."
- A single declarative fragment, no second clause: "Start with the problem."
- A question the section then answers.
- A sentence anchored in a specific, personal detail rather than an abstract contrast (see below).

## Sentence-level rules

- **Short, declarative, subject-verb-object.** Cut subordinate clauses. If a sentence needs a semicolon, it's two sentences.
- **Twin-sentence antithesis** is the signature move: state what it isn't, then what it is, in two short parallel fragments.
  > "The difference isn't the tool. It's the judgment behind it."
  > "Commercial judgement tells me what deserves to exist. Engineering lets me make it real."
- **No hedging language.** Not "could potentially," not "may help to." State the claim. If it's genuinely uncertain, say the uncertainty plainly ("small today, but the migration forecasts are not small") rather than softening with qualifiers.
- **First person is used to ground a claim in direct experience, not to editorialize.** "I keep hearing the same story from heads of payments at mid-tier banks" — evidence, not opinion.

## Structure (long-form: newsletter, blog, docs)

- Compact section headers that name an analytical lens, not a topic: "How the capture works," "What the regulators are saying," "The risks created by success" — not "Background" or "Overview."
- **Bold micro-labels inline** to segment an idea without a full subheading: "**The headline version:**", "**The card play:**"
- **Arrow chains** for causal or process explanation instead of prose paragraphs:
  > "User converts bank money -> stablecoin issued against reserve assets -> user holds token in a wallet that looks like a bank interface -> balance sheet has changed."
- One pull-quote block per piece, reserved for the sharpest line — not a summary sentence, a thesis sentence.
- **Close on a compressed, high-conviction restatement.** Two sentences, no qualifier, often the shortest sentences in the piece:
  > "The real threat is simpler and more serious: stablecoins threaten deposits. And deposits are where power lives."

## Use the career pivot as an actual asset, not a bio footnote

Fifteen years in enterprise FinTech sales, then learning to code from scratch in his forties, is a real differentiator — competitors writing about the same AI/FinTech topics generally have one half of that story or the other, not both. The default failure mode is leaving this out of the body copy entirely and parking it in a one-line sign-off. Let it surface inside the argument itself, not just the bio:

> "I spent fifteen years explaining technology. Then I learned to build it."

That's a stronger opening than an abstract industry claim precisely because no competitor can copy it. Reach for it when a piece is otherwise running on pure market analysis with no personal anchor — not in every piece, but more than once every few essays.

## Evidence rules

- Every number carries a source and a date: "Standard Chartered (Jan 2026): ~$500B could leave U.S. bank deposits by end of 2028."
- Never "experts say" or "studies show." Name the institution.
- Distinguish what's live from what's aspirational, explicitly. On the site this shows up as status labels (Live product / Production-ready repository / Private build / Pre-launch); in prose it's a plain sentence: "Those are funding-base numbers, not payments numbers."
- Claims are deliberately conservative. If something is unverified, say so rather than rounding up.

## The dry capstone

End a section with one short, understated line that lands the point without straining for a joke:

> "That is not whining. That is algebra."
> "That is not decentralisation. It is a new form of monetary concentration wearing a more fashionable jacket."

This is seasoning, not a running bit — one per section, max, and only when the line is actually sharp. Cut it before forcing it.

## Words

**Avoid:** revolutionary, cutting-edge, game-changing, seamless, robust (as filler), disruptive, unlock, leverage (as a verb for anything other than actual leverage), "in today's fast-paced world," any sentence that could appear in a press release for a company that doesn't exist yet.

**Prefer:** plain verbs, concrete nouns, specific numbers. If a sentence would read the same with the adjective removed, remove the adjective.

**Spelling:** British/international — judgement, organisation, optimise, colour. Not American.

**Punctuation:** No exclamation marks. No emoji in body copy. Em dashes and arrow notation (`->`) are fair game for compressing a causal chain.

## Bio / sign-off boilerplate

Use as-is or trim, don't embellish:

> Omar Al-Bakri builds AI systems for financial services. Previously 15 years in enterprise FinTech sales across Tier 1 banks and payments infrastructure.

## Before publishing, check

- [ ] Does the opening reframe something, or does it just announce a topic?
- [ ] **Count the "X is not Y" antithesis headlines/lines across the whole piece — is it more than one?** If yes, cut it down to one and rewrite the rest in a different shape.
- [ ] Any hedging words that can be cut?
- [ ] Every number sourced and dated?
- [ ] Any hype adjective that isn't earning its place?
- [ ] Does it end on a two-sentence thesis, not a trailing summary?
- [ ] Is the personal career-pivot story present anywhere, or has this piece gone fully abstract?
- [ ] British spelling throughout?
